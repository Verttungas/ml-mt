CRÍTICO 1. kNN — El tema estrella de la Dra. Villar ›
¿Por qué es crítico? La Dra. publicó papers sobre kNN ("Condensación controlada en k-NN"). Es SU tema. Tu compañero lo confirmó.

Concepto central

kNN clasifica un punto nuevo buscando los k vecinos más cercanos en los datos de entrenamiento y asignando la clase mayoritaria.

  • No paramétrico → no asume distribución de los datos
  • Lazy learning → no construye modelo, memoriza TODO el dataset
  • Supervisado → necesita datos etiquetados
  • k impar en problemas binarios → evita empates

Distancias — MEMORÍZALAS

Manhattan (L₁)Σ|xₖ - yₖ|m = 1
Euclidiana (L₂)√Σ(xₖ - yₖ)²m = 2

Trampa típica: Confundir * (multiplicación elemento a elemento) con %*% (multiplicación matricial) cuando se calcula Mahalanobis en R.

Función knn() en R — paquete class

library(class)
clasif = knn(
  train = CE[,1:2],  # solo columnas numéricas del CE
  test  = CP[,1:2],  # solo columnas numéricas del CP
  cl    = CE[,3],    # vector de clases del CE
  k     = 5,         # número de vecinos
  prob  = TRUE,      # guardar probabilidad del voto
  l     = 0,         # mínimo votos para decisión
  use.all = TRUE     # incluir todos los empatados en distancia k
)
Trampa clave del parámetro l: Si l=5 y k=8, las observaciones que no alcancen 5 votos de la clase ganadora → NA (duda). Si te preguntan "¿qué pasa si pones l=3?" → se vuelve más estricto, más NAs.
Trampa use.all: Con TRUE incluye TODOS los puntos empatados en la distancia k (puede ser más de k vecinos). Con FALSE selecciona aleatoriamente para mantener exactamente k.

Resubstitución vs. Retención

  • Resubstitución: test = CE (el mismo que train). Con k=1 SIEMPRE da 100% porque el punto más cercano a sí mismo es él mismo.
  • Retención: test = CP (datos no vistos). Da estimación realista.
Pregunta segura: "Si usas kNN con k=1 y test=train, ¿cuál es la clasificación correcta?" → 100% siempre.

Efecto de k en la frontera de decisión

  • k pequeño (ej: k=1) → frontera irregular, overfitting, sensible a ruido
  • k grande (ej: k=N) → frontera suave, underfitting, todo se clasifica como la clase mayoritaria

Matriz de confusión — "Muy sensible o muy específico"

Estimada Ω₀Estimada Ω₁
Real Ω₀TN (verdadero neg)FP (falso pos)
Real Ω₁FN (falso neg)TP (verdadero pos)
cc = (TP + TN) / (TP + TN + FP + FN) × 100
error = 1 − cc 
# En R:
mat = table(real = CP[,3], pred = clasif)
cc = sum(diag(mat)) / sum(mat) * 100
OJO: table(real, predicho) — el primer argumento son las clases reales, el segundo las predichas. Si los inviertes, la tabla se transpone y los FP/FN se intercambian.
CRÍTICO 2. Normalización / Feature Scaling ›

¿Cuándo normalizar?

  • Cuando el algoritmo usa distancias (kNN, K-means)
  • Cuando hay diferentes unidades de medida
  • Cuando hay grandes diferencias en rangos de valores
Dato clave del examen: En el dataset de vino, estandarizar mejoró de 71% → 95% porque las variables tenían rangos MUY diferentes (Pr: 278-1680 vs NFl: 0.13-0.66). En el dataset columna2C, NO mejoró porque los rangos ya eran similares.

Métodos de normalización

MétodoFórmulaResultado
Z-scorex_new = (x − μ) / σmedia=0, sd=1
Min-maxx_new = (x − x_min) / (x_max − x_min)rango [0,1]
Rangex_new = a + (x − x_min)(b − a) / (x_max − x_min)rango [a,b]
Logx_new = log(x) o log10(x)comprime valores grandes

En R: scale()

# Z-score completo (centra + escala):
datos[,1:4] = scale(datos[,1:4])

# Solo centrar (resta la media, NO divide por sd):
datos[,1:4] = scale(datos[,1:4], scale=FALSE)

# Verificar que sd = 1 después de estandarizar:
sapply(datos[-length(datos)], sd)  # debe dar 1
Trampa: scale(x, scale=FALSE) solo CENTRA (resta media), NO estandariza. Si te preguntan "qué pasa si quitas scale=TRUE" o "qué hace scale con scale=FALSE", la respuesta es que solo centra. Las sd originales se mantienen.
Otra trampa: NO debes estandarizar la columna de clases/etiquetas. Si tu dataframe tiene 5 columnas y la 5ta es la clase, solo haces scale(datos[,1:4]).

Naive Bayes vs Normalización

  • Estandarizar NO afecta a Naive Bayes (calcula media/sd internamente por clase)
  • Eliminar variables correlacionadas (>0.8) SÍ mejora NB porque viola el supuesto de independencia condicional

Clave: No hay regla fija, pero la más usada en sus prácticas es 70/30.

sample.split() — paquete caTools

library(caTools)
set.seed(55)  # SIEMPRE poner semilla para reproducibilidad
div = sample.split(datos$clase, SplitRatio = 0.7)
# div es vector booleano: TRUE = CE, FALSE = CP

CE = subset(datos, div == TRUE)
CP = subset(datos, div == FALSE)
Detalle crucial: sample.split() recibe la COLUMNA DE ETIQUETAS, no el dataframe completo. Conserva la proporción por clase (muestreo estratificado). Si le pasas el dataframe entero, no funciona como esperas.

Según tamaño de muestra

  • Muestra pequeña → Resubstitución (CE = CP). Rápido pero optimista.
  • Muestra grande → Retención (CE separado de CP). Más realista.
  • Muestra suficientemente grande → CE + CV + CP. Afina hiperparámetros con validación.
CRÍTICO 5. Trampas de Código R — "Quita un signo" ›
Formato de pregunta: Te dan código R y preguntan "¿qué da de output?" o "¿qué pasa si cambias X?"

Trampa 1: Operadores lógicos & vs &&

x = c(TRUE, FALSE, TRUE)
y = c(FALSE, TRUE, TRUE)

x & y   # → FALSE FALSE TRUE  (vectorizado, elemento a elemento)
x && y  # → FALSE  (solo evalúa el PRIMER elemento)

Trampa 2: * vs %*%

A = matrix(1:4, nrow=2)   # |1 3|
B = matrix(5:8, nrow=2)   # |2 4|

A * B    # multiplicación ELEMENTO a ELEMENTO
A %*% B  # multiplicación MATRICIAL real

Trampa 3: matrix() llena por COLUMNA

m = matrix(1:6, nrow=2, ncol=3)
# Resultado:      |1 3 5|
#                  |2 4 6|
# NO es:          |1 2 3|
#                  |4 5 6|

# Para llenar por renglón:
m = matrix(1:6, nrow=2, ncol=3, byrow=TRUE)
# Resultado:      |1 2 3|
#                  |4 5 6|

Trampa 4: Índices negativos = EXCLUIR

v = c(10, 20, 30, 40, 50)
v[3]     # → 30 (posición 3)
v[-3]    # → 10 20 40 50 (TODO EXCEPTO posición 3)
v[c(-1,-2)] # → 30 40 50 (todo excepto 1 y 2)

Trampa 5: Reciclaje de vectores

x = c(1, 2, 3, 4, 5, 6)
y = c(10, 20)
x + y  # → 11 22 13 24 15 26  (y se recicla: 10,20,10,20,10,20)

Trampa 6: na.rm = TRUE

v = c(1, 2, NA, 4)
sum(v)                # → NA  (¡sin na.rm!)
sum(v, na.rm=TRUE)    # → 7   (ignora el NA)
mean(v, na.rm=TRUE)   # → 2.333  (7/3, NO 7/4)
OJO: mean() con na.rm=TRUE divide entre el número de valores NO-NA (3), no entre el total (4).

Trampa 7: Precedencia de operadores

# ^ → unario -,+ → %% → *,/ → +,- → comparaciones → ! → &,&& → |,||
2 + 3 * 4     # → 14 (no 20)
-2 ^ 2        # → -4 (no 4, porque ^ tiene mayor prioridad que -)
!TRUE & FALSE # → FALSE  (! se evalúa primero → FALSE & FALSE)

Trampa 8: ifelse() vectorizado vs if/else

x = c(1, -2, 3, -4)
ifelse(x > 0, "pos", "neg")  # → "pos" "neg" "pos" "neg" (vectorizado)

# if/else NO es vectorizado, solo evalúa el primer elemento:
if(x > 0) "pos" else "neg"  # → "pos" (solo evalúa x[1]=1)
ALTO 6. Los 6 Valores — Rome Call + IA Responsable ›

Los 6 principios del Rome Call for AI Ethics

#PrincipioSignifica que...
1TransparenciaLos sistemas de IA deben ser comprensibles para todos
2InclusiónNo discriminar a nadie; todo ser humano tiene igual dignidad
3Responsabilidad (Accountability)Siempre debe haber alguien que responda por lo que hace la máquina
4ImparcialidadLos sistemas no deben seguir ni crear sesgos
5Fiabilidad (Reliability)La IA debe ser confiable
6Seguridad y PrivacidadLos sistemas deben ser seguros y respetar la privacidad
Mnemónico: T-I-R-I-F-S → "TIRIFS" o piénsalo como: Transparencia, Inclusión, Responsabilidad, Imparcialidad, Fiabilidad, Seguridad.

ISO: IA Ética vs IA Responsable

  • Ética = valores / intención (brújula moral)
  • Responsable = controles / herramientas prácticas (mapa + equipo)

3 preocupaciones éticas ISO: sesgo, uso indebido de datos personales, falta de transparencia.

3 líneas de defensa: 1ª operativa (devs), 2ª supervisión (riesgos/compliance), 3ª independiente (auditoría).

Funciones base R

FunciónQué hace
mean(x)Media aritmética
median(x)Mediana
var(x)Varianza muestral
sd(x)Desviación estándar muestral
quantile(x)Cuartiles (0, 25, 50, 75, 100%)
IQR(x)Rango intercuartílico (Q3 − Q1)

Trampa: range(x) devuelve c(min, max), NO el valor del rango. Para obtener el rango numérico necesitas diff(range(x)).

Requieren paquetes o cálculo manual

  • Moda: DescTools::Mode(x) o modeest::mfv(x)
  • Asimetría: moments::skewness(x)
  • Curtosis: moments::kurtosis(x)
  • CV: sd(x)/mean(x)*100 (manual)
Notación paquete::funcion() permite usar una función sin cargar toda la librería con library().

Interpretaciones para examen

  • Sesgo > 0 = asimetría derecha (cola a la derecha, media > mediana)
  • Sesgo < 0 = asimetría izquierda (cola a la izquierda, media < mediana)
  • Curtosis > 3 = leptocúrtica (colas pesadas, más puntiaguda)
  • CV menor = mayor consistencia entre datos

Fórmulas de normalización

Z-score: x_new = (x − μ) / σ → media=0, sd=1

Min-max: x_new = (x − x_min) / (x_max − x_min) → [0,1]

Range [a,b]: x_new = a + (x − x_min)(b − a) / (x_max − x_min)

Distancias (Minkowski general: (Σ|xₖ-yₖ|^m)^(1/m))

Manhattan: m=1 → Σ|xₖ-yₖ|

Euclidiana: m=2 → √Σ(xₖ-yₖ)²

Chebyshev: m→∞ → max|xₖ-yₖ|
